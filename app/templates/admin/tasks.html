📡 Collecte RSS

Déclenche immédiatement la collecte de tous les flux RSS actifs. Normalement exécutée toutes les heures automatiquement.

📝 Synthèses quotidiennes

Génère les synthèses IA pour tous les utilisateurs et catégories. Choisissez la date cible (par défaut : hier, J-1).

ℹ️ Le déclenchement manuel régénère toujours les synthèses existantes (force=true).

📊 Synthèses hebdomadaires

Génère la super-synthèse des 7 derniers jours glissants (J-7 → hier) avec faits marquants, tendances et Cyber Brief LinkedIn. Prend toutes les synthèses quotidiennes disponibles, même si certains jours manquent.

ℹ️ Le déclenchement manuel régénère toujours les synthèses existantes (force=true).

📧 Envoi emails quotidiens

Envoie les emails de synthèse quotidienne aux utilisateurs abonnés. Nécessite que les synthèses aient été générées au préalable.

🔍 Enrichissement full-text des articles

Visite les articles dont le contenu est trop court (< 500 caractères) et extrait le texte complet depuis la page source via newspaper3k. Exécuté automatiquement 30 min après chaque collecte RSS.

(par exécution)
ℹ️ Rate limiting : 2s entre chaque requête par domaine. Les sites bloqués (Cloudflare, paywall) sont ignorés sans régression.

📨 Envoi emails hebdomadaires

Envoie la super-synthèse de la semaine précédente avec faits marquants, tendances et Cyber Brief LinkedIn. Le renvoi est forcé par défaut.

ℹ️ Concerne les 7 derniers jours glissants (J-7 → hier). Prend toutes les synthèses quotidiennes disponibles. Le renvoi est forcé même si l’email a déjà été envoyé.

🩺 Vérification de santé

Vérifie que les synthèses de J-1 ont bien été générées et les emails envoyés. Envoie un email d'alerte à ADMIN_EMAIL si une anomalie est détectée. Planifiée automatiquement à 10h00 chaque jour.

ℹ️ Configurer HEALTH_CHECK_ALWAYS_REPORT=true dans le .env pour recevoir le rapport même sans anomalie.

🗑️ Supprimer des synthèses par date

Supprime définitivement les synthèses générées un jour précis. Utile pour forcer une régénération propre sans doublon.

🔄 Réinitialiser la date de collecte des flux

Remet à zéro le champ last_fetched des flux sélectionnés. La prochaine collecte récupérera tous les articles disponibles dans le flux, sans limite de date. Utile après un bug de collecte ou pour récupérer des articles manquants.
